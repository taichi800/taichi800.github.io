Service Scope, Non-Features, and Change Triggers

Last updated: 2025-08-12

  • Privacy and Device Access
  • Terms of Service

Current status

This site is static and content-only. The following statements are true today:

  • No user accounts and no profiles.
  • No comments, forums, messaging, uploads, or chats.
  • No multi-site search. Any search, if present, is limited to this site only.
  • No advertising, affiliate tracking, or referral pixels.
  • No analytics, session replay, heatmaps, or A/B testing.
  • No non-essential cookies, no localStorage, no IndexedDB, no fingerprinting.
  • No third-party embeds by default. Fonts, scripts, and media are hosted locally.
  • No payments, orders, donations with perks, or subscriptions sold through this site.
  • No adult or age-restricted content.
  • No collection of location, microphone, camera, or biometric data.
  • No push notifications, background sync, or installed PWA behaviors that store data.
  • No public API endpoints that accept user input.
  • Hosting is on GitHub Pages. Operational logs are handled by GitHub under its own policies.

Scope line: No user-to-user features. No multi-site search. No non-essential cookies or persistent storage.

Services not offered

To avoid confusion, each item below is not available on this site:

  • Account registration.
  • User login.
  • User profiles or avatars.
  • Public comments.
  • Message boards or forums.
  • Direct messages or chats.
  • User uploads of files, images, audio, or video.
  • Live streams or live events.
  • Multi-site or web-wide search.
  • Advertising placements.
  • Affiliate links that track users.
  • Analytics, session replay, or heatmap scripts.
  • Preference cookies or saved settings across visits.
  • Third-party widgets that set tracking cookies.
  • Payments, orders, or subscriptions.
  • Age-restricted or adult areas.
  • Collection of precise location, audio, video, or biometrics.
  • Push notifications, background sync, or offline caching beyond normal browser caching.
  • Public API keys or interactive endpoints.

Change triggers and required actions

If any trigger becomes true in the future, we will update the Privacy and Terms pages before enabling the feature. Each trigger has a matching action list.

Accounts and identity

  • Trigger: add registration, login, usernames, or profiles.
  • Actions: describe data collected, legal bases, retention, deletion; add account rules, acceptable use, suspension and appeals; add age gates if needed.
  • Trigger: enable Single Sign-On or federated login.
  • Actions: name providers, list data received, note cross-border transfers; add user responsibility for SSO security.

User submissions and interaction

  • Trigger: enable comments, forums, chats, messaging, uploads, or automatic PR-to-publish.
  • Actions: add content standards, license grant, moderation powers, takedown policy, reporting routes; describe processing of UGC and abuse reports.
  • Trigger: expand mail-to-publish beyond curated, manual publication.
  • Actions: clarify selection, editing, retention, withdrawal; confirm non-exclusive publication license and removal process.

Cookies, local storage, and device access

  • Trigger: any non-essential storage or access, for example analytics, A/B tests, remembering time zone or preferences.
  • Actions: add a Cookies and similar technologies section with purposes, providers, durations; deploy a consent banner with equal Accept and Reject; block non-essential scripts until consent.
  • Trigger: bot-defense or CAPTCHA that fingerprints devices.
  • Actions: describe data points read, provider, retention; justify as strictly necessary or obtain consent.

Analytics, measurement, and profiling

  • Trigger: enable page analytics, error tracking tied to users, heatmaps, or session replay.
  • Actions: identify processor, IP handling, retention, opt-out; obtain consent if not strictly necessary; consider a DPIA for rich tracking.

Marketing and contact

  • Trigger: start a newsletter or direct marketing.
  • Actions: capture consent, provide one-click unsubscribe, maintain suppression lists; name provider; state retention and frequency.
  • Trigger: add affiliate links or ads.
  • Actions: disclose partners and tracking; obtain consent for any pixels; state independence and labeling rules.

Payments, sales, or subscriptions

  • Trigger: sell digital or physical goods, accept donations with perks, or run subscriptions.
  • Actions: add prices, taxes, delivery, refund and cooling-off terms; payment processor details; cancellation method; fraud checks and retention.

Sensitive or age-restricted content

  • Trigger: host adult or other restricted content.
  • Actions: implement age checks; add eligibility and access restrictions; describe age-assurance data and deletion timelines.

Location, biometric, or audio features

  • Trigger: collect or infer location, use microphone or camera, accept voice input, or process biometrics.
  • Actions: add explicit categories collected, consent, purpose, retention, on-device versus server processing; add environment safety notes in Terms.

Third-party embeds and processors

  • Trigger: embed maps, videos, comments, fonts from CDNs, or social widgets.
  • Actions: list each third party and data flow; load only after click or consent; add disclaimers for third-party outages.
  • Trigger: add new vendors or move hosting.
  • Actions: update processor list, locations, and transfer mechanisms; update availability language.

Technical changes that affect data

  • Trigger: add service worker, PWA install, background sync, or push notifications.
  • Actions: explain what is cached and for how long; provide instructions to clear; add opt-out.
  • Trigger: publish API endpoints or downloadable software.
  • Actions: document API logging, rate limits, license, reverse engineering rules, and any export controls.

Legal and jurisdictional scope

  • Trigger: target or market to a new country or state with special rules.
  • Actions: add that jurisdictionâ€™s disclosures and rights; update governing law or consumer notices where mandatory.

Security events and policy posture

  • Trigger: data breach, significant incident, or new high-risk processing.
  • Actions: add breach notice practices, update security measures, adjust retention; notify authorities where required.

Children and education contexts

  • Trigger: direct content to children or run school programs.
  • Actions: add parental consent flows, data minimization, guardian contact route; update eligibility terms.

Intellectual property policy

  • Trigger: accept user works, host fonts, or change licensing of site assets.
  • Actions: update content licenses, attribution requirements, hotlink rules, trademark usage, and DMCA notice procedure.

Business changes

  • Trigger: merger, acquisition, or transfer of operation.
  • Actions: add transfer of control clause and user rights at transfer; update successor language in Terms.

Federated presence or hosting

  • Trigger: operate a federated server or enable cross-instance posting.
  • Actions: document moderation logs, federation metadata, and partners; add federation conduct code and defederation rights.

Automated decision-making

  • Trigger: use filters, classifiers, or automated moderation that materially affects publication or visibility.
  • Actions: disclose logic in meaningful terms, significance, consequences, and provide a way to contest.

Pre-deployment gate

Before launching any change, we run this checklist:

  • Will anything be stored on or read from the device beyond what is strictly necessary for the feature the user asked for.
  • Will any user content be visible to any other user without manual editorial review.
  • Will any new third party receive data from visitors, including via an embed.
  • Will we collect contacts, billing, location, audio, video, biometrics, or identifiers.
  • Will we sell, accept donations with perks, or run subscriptions.
  • Are we targeting a new jurisdiction or an audience such as children.
  • Are we changing where or how we host or process data.
  • Are we changing licenses or IP terms for site content.

If any answer is yes, we update the Privacy and Terms pages before launch and state the effective date.

Advance notice and versioning

If we introduce accounts, comments, forums, third-party embeds, analytics, advertising, sales, or any other feature that changes how your information is processed or how you interact with others, we will update the Privacy and Terms pages before launch and set a new effective date. We will not enable non-essential cookies or persistent storage without your explicit choice.
